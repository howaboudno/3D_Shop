Standing rules — every phase, no exceptions

  • Deploy at the end of every phase. A phase that only runs on localhost is not finished.
  • Alembic from commit one. Postgres (D16) means no more Railway shell patches.
  • Integer cents everywhere (D4). Format at render only. No float touches money, including in JSON.
  • One authorization dependency, written once. get_order_for_user() — buyer sees own, seller sees all.
  • The API never touches file bytes (D15). If you write await file.read(), stop.
  • Record grams_used on every order even though v1 charges nothing for it (§5.1). That is the whole migration path to per-gram.
